Resolvidos 10 itens

  • ✓ LP iFood — cores de títulos e links Movi os estilos de cor dos títulos e links da LP iFood para page-ifood.css. As cores laranja dos headings e dos links de parceiro continuam iguais visualmente. O arquivo é carregado condicionalmente apenas no template page-ifood.php.
  • ✓ LP iFood — animação marquee Extraí a animação de rolagem infinita do marquee para components/marquee.css. O keyframe scroll-left e os estilos do wrapper agora vivem em arquivo externo, carregado só na LP iFood. Componente reutilizável para a página de regras também.
  • ✓ Produto detalhe — estilos de impressão Criei print-produto-detalhe.css com os estilos de tabela para impressão. O modal de impressão agora referencia o arquivo via tag link em vez de injetar CSS inline no document.write. Layout de impressão preservado sem alterações visuais.
  • ✓ Galeria — scrollbar customizado Movi o scrollbar laranja personalizado para utils/scrollbar.css, um utilitário compartilhado. Ele é carregado na página de galeria e será reaproveitado em outras seis páginas que usam o mesmo estilo de scrollbar com as cores da marca Seara.
  • ✓ Header — menu logado e sub-menus Transferi os estilos de visibilidade do menu logado e sub-menus para header.css, carregado globalmente. As transições de opacity e visibility no hover continuam funcionando. Inclui o media query para sub-menus em telas até 1024px.
  • ✓ Admin — meta box CTA iFood Extraí os estilos da meta box de Call to Action do iFood para admin-cta-metabox.css. O layout de tabela, inputs e textareas do painel admin continua idêntico. O arquivo é enfileirado via admin_enqueue_scripts em post.php e post-new.php.
  • ✓ Header — OneTrust wrapper Movi a função OptanonWrapper vazia exigida pelo OneTrust para vendor/onetrust-wrapper.js. O script é carregado no head via wp_enqueue_script em todas as páginas, garantindo que o callback esteja disponível quando o SDK do OneTrust inicializar.
  • ✓ Header Gourmet Gradina — remoção de script vazio Removi a tag script completamente vazia que existia no componente HeaderGourmetComponent.php do tema Gradina. Não havia nenhum código dentro dela, então a remoção não causa impacto funcional. Uma dependência inline a menos no carregamento.
  • ✓ Header Gradina — remoção de script vazio Removi a tag script vazia do HeaderComponent.php do tema Gradina, mesmo caso do header gourmet. Sem código dentro, sem impacto funcional. Com essa remoção, os dois headers do Gradina ficam livres de tags script desnecessárias no HTML.
  • ✓ Single material — estilos de headings e breadcrumb Criei single-material.css com os estilos de cor e tamanho dos headings e a ocultação de itens do breadcrumb. Todos os seletores foram escopados com .single-material para não vazar para outras páginas. Carregado via is_singular('material').
